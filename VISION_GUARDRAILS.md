# Pioneer Vision Guardrails

These principles define the identity of Pioneer.

Every design, UX, and technical decision should be evaluated against these rules.

---

## G001 — Image First

The captured image is always the center of the experience.

No workflow should completely hide the image unless absolutely necessary.

---

## G002 — Session Oriented

Pioneer is organized around the astrophotography session.

Not around individual devices.

Users think in tasks, not hardware.

---

## G003 — Mobile First

Every feature must be designed for smartphone usage before considering larger screens.

Tablet and desktop layouts are secondary adaptations.

---

## G004 — One-Hand Operation

The application should be comfortably usable with one hand.

Critical actions must remain accessible without complex gestures.

---

## G005 — Glove Friendly

Interactive elements must remain usable during field operation.

Buttons and touch targets should prioritize reliability over density.

---

## G006 — Minimal Cognitive Load

The interface should reduce decision making.

Only relevant information should be visible at any given moment.

---

## G007 — Two-Tap Rule

Critical actions should never require more than two taps.

---

## G008 — Contextual Workspace

The interface adapts to the current stage of the session.

The user should not navigate through multiple unrelated screens.

---

## G009 — Information Over Configuration

Monitoring information is more important than settings.

Configuration should stay out of the way during operation.

---

## G010 — No Desktop Remoting Mentality

Pioneer is not a remote desktop.

Every interaction should be designed specifically for touch devices.

---

## Core Principle

Every interaction should keep the user focused on the sky, not on the software.
